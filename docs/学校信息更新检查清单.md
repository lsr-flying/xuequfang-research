# 学校信息更新检查清单（2026年版）

## 📋 更新原则与标准

### 核心原则
1. **时效性优先**: 所有数据更新到2026年4月
2. **历史对比**: 保留2024-2026年关键数据对比
3. **权威来源**: 标注信息来源和可靠性等级
4. **专业分析**: 加入趋势预测和风险评估
5. **用户友好**: 结构清晰，便于阅读和决策

### 更新时间要求
- **政策文件**: 每年3-4月招生季更新
- **房价数据**: 每月更新市场价，每季度深度分析
- **学校数据**: 每年9月新学期更新在校生数据
- **家长反馈**: 每学期收集更新
- **全面检查**: 每年1月全面检查所有文件

## ✅ 元数据检查清单

### 文件头信息检查
- [ ] `title`: 学校名称准确完整
- [ ] `type`: 学校类型正确（公办小学/初中/民办等）
- [ ] `zone`: 所属区域准确（嘉定区-XX镇）
- [ ] `tier`: 梯队等级正确（第一/第二/第三梯队）
- [ ] `tags`: 包含[对口, 预警, 历史对比]等标签
- [ ] `updated`: 更新为"2026-04"
- [ ] `source`: 标注信息来源（官方文件/权威媒体/家长反馈）
- [ ] `reliability`: 标注可靠性等级（高/中/低）

### 元数据标准
```yaml
---
title: 学校名称
type: 公办小学/公办初中/民办小学/民办初中
zone: 嘉定区-XX镇
tier: 第一梯队/第二梯队/第三梯队
tags: [对口, 预警, 历史对比, 特色标签]
updated: 2026-04
source: 官方文件/权威媒体/家长反馈
reliability: 高/中/低
---
```

## ⚠️ 常见问题与解决方案

### 数据缺失问题
| 问题类型 | 解决方案 | 备注 |
|----------|----------|------|
| **2026年官方数据未发布** | 使用"预计"、"预测"标注，说明数据来源 | 避免绝对化表述 |
| **历史数据不完整** | 标注"数据缺失"，基于趋势合理估算 | 保持透明度 |
| **房价数据不一致** | 取多个来源平均值，标注数据来源 | 说明可能存在误差 |
| **学校数据不公开** | 基于往年数据和合理增长估算 | 标注为估算值 |

### 更新时效性问题
| 问题类型 | 解决方案 | 更新时间 |
|----------|----------|----------|
| **政策文件** | 每年3-4月招生季更新 | 强制更新 |
| **房价数据** | 每月更新市场价 | 建议更新 |
| **学校动态** | 每学期更新一次 | 建议更新 |
| **家长反馈** | 每年收集一次 | 建议更新 |

### 质量控制问题
| 问题类型 | 质量控制措施 | 负责人 |
|----------|----------------|--------|
| **数据准确性** | 交叉验证，多来源比对 | 更新人员 |
| **分析客观性** | 多人审核，避免主观偏见 | 审核人员 |
| **格式统一性** | 使用模板，定期检查 | 维护人员 |
| **链接有效性** | 定期检查，修复死链 | 维护人员 |

## 📅 更新计划与时间表

### 年度更新计划
| 更新时间 | 更新内容 | 优先级 | 负责人 |
|----------|----------|--------|--------|
| **每年1月** | 全面检查所有文件 | 高 | 维护团队 |
| **每年3月** | 更新政策文件 | 最高 | 政策组 |
| **每年4月** | 更新招生信息 | 最高 | 学校组 |
| **每年6月** | 更新房价数据 | 高 | 房产组 |
| **每年9月** | 更新学校数据 | 中 | 学校组 |
| **每年12月** | 年度总结报告 | 中 | 分析组 |

### 月度维护计划
| 维护内容 | 频率 | 检查要点 |
|----------|------|----------|
| **房价数据** | 每月 | 市场价变化，成交量 |
| **政策动态** | 每月 | 教育局通知，政策变化 |

| **家长反馈** | 每季度 | 论坛，社群，访谈 |

| 触发条件 | 响应时间 | 更新内容 |
|----------|----------|----------|
| **重大政策调整** | 24小时内 | 政策文件，相关分析 |

| **市场剧烈波动** | 72小时内 | 房价分析，购房建议 |
